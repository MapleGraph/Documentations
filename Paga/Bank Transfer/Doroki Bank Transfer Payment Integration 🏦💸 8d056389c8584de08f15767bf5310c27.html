üìö Overview

This feature enables users to accept payments through bank transfers. In this document, we will cover the following aspects:

  • Workflow and Process
  • Setup and Configuration
  • Testing and Validation
  • API Documentation

üîç Workflow and Process

  1. The user clicks on the "Bank Transfer" payment mode during checkout.
  1. The system checks if the "Bank Transfer" payment configuration is available.
  1. If the payment configuration for the "Bank Transfer" mode is available, the payment process is initiated.
  1. To start the process, the system calls the "secured/initiateBankTransferPayment" API, passing the amount and terminalId as parameters. (See the API documentation for more details.)
  1. The system receives the bank account number in the response from the API.
  1. The bank account number is displayed on the "Check Payment Status" page for the user to initiate the bank transfer.
  1. The user initiates the bank transfer from their banking portal (e.g., Paga dashboard) or visits their bank branch to complete the transfer, using the provided bank account number.
  1. The user can manually check the payment status by clicking the "Check Status" button on the "Check Payment Status" page.
  1. Once the bank transfer is detected and matched successfully, the payment is marked as successful, and the order is closed.

Test Cases

  • Verify that flow integration is similar to the flow shared from design team.
  • Verify that UI components are visibly similar to design prototype.
  • Test error handling scenarios, such as invalid reference numbers or transfers with incorrect amounts.
  • Verify that the user's order status is updated correctly after a successful transfer detection.
