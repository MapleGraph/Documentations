üìö Overview

This feature enables users to accept payments through bank transfers. In this document, we will cover the following aspects:

  • Workflow and Process
  • Setup and Configuration
  • Testing and Validation
  • API Documentation

üîç Workflow and Process

  1. The merchant selects the "USSD" payment option during checkout.
  1. The system verifies the availability of the "USSD" payment configuration.
  1. If the configuration is available, the system initiates the payment process by calling the "/getBankUSSDPaymentBanks" API with the businessName parameter.
  1. The API responds with a list of banks supporting USSD payments.
  1. The merchant selects the bank provided by the consumer from the displayed bank list.
  1. The system calls the "secured/initiateBankUSSDPayment" API, passing the amount, terminalId, and the chosen bank's publicId.
  1. The API responds with a unique USSD code.
  1. The USSD code is provided to the consumer, who enters it on their mobile phone and follows the prompts to approve the payment.
  1. On the same page, the merchant can check the payment status by clicking the "Check Status" button.
  1. The "Check Status" button triggers a call to the "secured/checkDynamicBankTransferPayment" API with the paymentRequestId and amount.
  1. The API returns the payment status as either "Payment completed" or "Payment pending".
  1. If the status is "Payment pending", the page remains unchanged.
  1. If the status is "Payment completed", the order is marked as complete.

üß™ Testing and Validation

Test Cases

  • Verify that the system correctly checks for the availability of the USSD payment configuration.
  • Test the behavior when the USSD payment configuration is not available (error message should be displayed, and the payment process should be terminated).
  • Verify that the list of banks is correctly displayed on the Bank List Page.
  • Test error handling scenarios, such as invalid reference numbers or transfers with incorrect amounts.
  • Verify that the user's order status is updated correctly after a successful transfer detection.

Testing Process

To complete the USSD payment testing process, follow these steps:

  1. Initiate USSD Payment from the App
    • Initiate a USSD payment from the application.
    • Choose any bank for testing purposes.
  1. Obtain the USSD Code
    • After initiating the payment, you will receive a USSD code on the "Check Status" page.
    • The USSD code will follow the format: "*320*000*724+67429891#"
  1. Extract Customer Code and Short Code
    • From the USSD code, extract the 8-digit customer code after the "+" sign and before the "#" sign (e.g., 67429891).
    • The first three digits of the USSD code represent the short code (e.g., 320).
  1. Call the "getDetails" API
    • Run the following cURL request, replacing the customerRef and shortCode with the values extracted from the USSD code:
      curl --location '<https://qa1.mypaga.com/paga-webservices/coralPay/secured/getDetails>' \\
     --header 'Content-Type: application/json' \\
     --header 'Authorization: Basic RjI1MjkyNjEtQ0M0RS00MzEzLTg0OTktOTJCMkU3RTRCRkQ4OlBhc3N3b3JkMQ==' \\
     --data '{ "merchantId": "1057PGA10000001", "customerRef": "67429891", "shortCode": "329" }'
    • Expected Response:
      {
  "customerName": "Joe Leo",
  "amount": 121.0,
  "displayMessage": "Payment for Joe Leo",
  "responseCode": "00",
  "traceId": "4C86EDF6-771E-EF11-86D2-BF9ADD20706C_1717068948977"
}
    • Note the traceId value from the response.
  1. Call the "paymentNotification" API
    • Run the following cURL request, replacing the traceId, customerRef, and paymentReference with the appropriate values:
      curl --location '<https://qa1.mypaga.com/paga-webservices/coralPay/secured/paymentNotification>' \\
     --header 'Content-Type: application/json' \\
     --header 'Authorization: Basic RjI1MjkyNjEtQ0M0RS00MzEzLTg0OTktOTJCMkU3RTRCRkQ4OlBhc3N3b3JkMQ==' \\
     --header 'Cookie: route=1717068949.232.3130.280750|54a86a183ec676f78c6564f15a1b629b; paga-webservices-canary=always' \\
     --data '{ "paymentReference": "87900590274750678099184958767864", "customerRef": "67429891", "responseCode": "00", "merchantId": "1057PGA10000001", "mobileNumber": "0809***7503", "amount": "121.00", "passBackReference": "e19f2d56-c420-4625-9ea5-cf1cf735b0e3", "transactionDate": "2020-10-26T22:22:23.2987732+01:00", "billerCode": null, "billerPrefix": null, "traceId": "4C86EDF6-771E-EF11-86D2-BF9ADD20706C_1717068948977", "shortCode": "889", "currency": "NGN", "Channel": "USSD", "hash": "9b03ebca981ab485a572d2a36d7a97051d1a2dba9ad5515c720decd3de817ae4" }'
    • Expected Response: 200 Success
  1. Check Payment Status
    • Upon receiving a successful (200) response from the "paymentNotification" API, you can check the payment status in the application.
    • The payment should be marked as successful.

By following these testing steps, you can emulate the USSD payment process and validate the functionality of the feature.
